# 🚀 GigShield: AI-Powered Parametric Insurance for Gig Workers

## 📌 About the Project
GigShield is an AI-based insurance platform designed for delivery workers like Swiggy and Zomato partners. It protects their income when they are unable to work due to external disruptions like rain, heat, or pollution by automatically providing compensation.

## 💡 Inspiration
Delivery workers depend on daily earnings, and even one bad day can affect their income. We wanted to build a solution that supports them during such situations.

## ⚙️ What it does
Workers can register and get a weekly insurance plan based on their risk level. The system monitors conditions like weather, and if a disruption occurs, it automatically triggers a claim and provides a payout.

## 🛠️ How we built it
We used React for the frontend and planned a Node.js backend. We designed a dashboard, premium calculator, and claim system, and planned to integrate weather APIs and AI for risk calculation.

## ⚠️ Challenges we ran into
Understanding parametric insurance and designing automatic claim triggering was challenging. We also faced setup issues and difficulty simulating real-world data.

## 🏆 Accomplishments that we're proud of
We built a structured solution for a real problem and created a working frontend with a clear workflow and features.

## 📚 What we learned
We learned about parametric insurance, improved our React skills, and understood how AI can be applied in real-world systems.

## 🔮 What's next for GigShield
We plan to build the backend, integrate real APIs, improve AI models, and add fraud detection to make the system more realistic.
